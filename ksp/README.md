# kotlin-behave KSP Annotation Processor

Compile-time generation of step definition interfaces from `.feature` files.
No string matching at runtime â€” full IDE navigation and type safety.

---

## How it works

1. Annotate your step definitions class with `@BehaveFeature`
2. KSP reads the `.feature` file at build time and generates a `*Spec` interface
3. Implement the generated interface â€” the compiler tells you if a step is missing
4. Pass the generated `steps()` factory to `gherkin()`

---

## Setup

### 1. Add dependencies

```kotlin
// build.gradle.kts of the module containing your step definitions

plugins {
    id("com.google.devtools.ksp") version "2.1.20-1.0.32"
}

dependencies {
    // Annotations â€” compile-only, KMP-compatible
    compileOnly(project(":annotations"))          // or the published artifact
    // KSP processor â€” runs at build time on JVM only
    ksp(project(":ksp"))                          // or the published artifact
}

ksp {
    // Path relative to project root where .feature files live.
    // Default: "src/commonTest/resources"
    arg("behave.featureDir", "src/commonTest/resources")
    arg("behave.projectDir", projectDir.absolutePath)
}
```

### 2. Annotate your class

```kotlin
@BehaveFeature("features/login.feature")
class LoginSteps(val compose: ComposeUiTest) : LoginStepsSpec {

    override suspend fun givenIAmOnTheLoginPage() {
        compose.onNodeWithTag("login_screen").assertIsDisplayed()
    }

    override suspend fun whenIEnterAsUsername(string: String) {
        compose.onNodeWithTag("username_field").performTextInput(string)
    }

    // ... implement all generated methods
}
```

### 3. Wire to the runner

```kotlin
gherkin("features/login.feature", LoginStepsSpec.steps { LoginSteps(compose) })
```

## Generated interface

Given `features/login.feature`:

```gherkin
Feature: Login

  Background:
    Given I have a collection {string}

  Scenario: Successful login
    Given I am on the login page
    When I enter {string} as username
    Then I see the dashboard

  Scenario Outline: Login as <role>
    Given I am logged in as <role>
    Examples:
      | role  |
      | admin |

  Scenario: Multiple cats
    Given the following cats:
      | name   | color | age |
      | Whisky | black | 3   |
```

KSP generates:

```kotlin
// Generated by kotlin-behave KSP â€” do not edit
package com.example.steps

interface LoginStepsSpec {

    suspend fun givenIHaveACollection(string: String)
    suspend fun givenIAmOnTheLoginPage()
    suspend fun whenIEnterAsUsername(string: String)
    suspend fun thenISeeTheDashboard()
    suspend fun givenIAmLoggedInAs(role: String)           // <role> â†’ String
    suspend fun givenTheFollowingCats(rows: List<GivenTheFollowingCatsRow>)

    companion object {
        fun steps(factory: () -> LoginStepsSpec): StepDefinitions<LoginStepsSpec> =
            io.mcol.behave.steps.steps(factory) {
                Given("I have a collection \\{string\\}") { (s: String) ->
                    ctx.givenIHaveACollection(s)
                }
                // ...
            }
    }
}

data class GivenTheFollowingCatsRow(val name: String, val color: String, val age: String)
```

## `@BehaveType` â€” custom parameter types

Override how `{placeholder}` tokens or DataTable columns map to Kotlin types.

### Placeholder mode

Maps a single `{placeholder}` token to a type:

```kotlin
@BehaveFeature("features/login.feature")
@BehaveType(placeholder = "label", type = ButtonLabel::class)
class LoginSteps : LoginStepsSpec
```

Step `And I click the {label} button` generates:
```kotlin
suspend fun andIClickTheButton(label: ButtonLabel)
```

### Field auto-detect mode

Absorbs all step tokens whose names match the primary constructor parameters of a type:

```kotlin
data class Credentials(val username: String, val role: String)

@BehaveFeature("features/login.feature")
@BehaveType(type = Credentials::class)   // absorbs {username} + {role}
class LoginSteps : LoginStepsSpec
```

Step `When I enter {username} as {role} with {int} attempts` generates:
```kotlin
suspend fun whenIEnterAsWithAttempts(credentials: Credentials, int: Int)
```

### Field explicit mode

Same as auto-detect but with column/placeholder names supplied explicitly:

```kotlin
@BehaveType(type = Credentials::class, fields = ["username", "role"])
```

### DataTable mode

Works identically for DataTable columns:

```kotlin
data class Cat(val name: String, val color: String)

@BehaveFeature("features/cats.feature")
@BehaveType(type = Cat::class)               // auto-detect: columns name + color
@BehaveType(placeholder = "age", type = CatAge::class)   // column "age" â†’ CatAge
class CatSteps : CatStepsSpec
```

Step with a DataTable having columns `name | color | age` generates:
```kotlin
data class GivenTheFollowingCatsRow(
    val cat: Cat,    // columns: name, color
    val age: CatAge  // column: age
)

suspend fun givenTheFollowingCats(rows: List<GivenTheFollowingCatsRow>)
```

## Method name generation

| Step text | Generated method name |
|---|---|
| `Given I am on the login page` | `givenIAmOnTheLoginPage` |
| `When I enter {string} as username` | `whenIEnterAsUsername` |
| `Then I have {int} items` | `thenIHaveItems` |
| `Given I am logged in as <role>` | `givenIAmLoggedInAs` |
| `And I click the {label} button` | `andIClickTheButton` |
| `Given the following cats:` | `givenTheFollowingCats` |

Two steps that produce the same name get numeric suffixes: `givenIHaveItems0`, `givenIHaveItems1`.

## KSP processor option reference

| Option | Default | Description |
|---|---|---|
| `behave.featureDir` | `src/commonTest/resources` | Root directory for `.feature` files, relative to `behave.projectDir` |
| `behave.projectDir` | `.` | Absolute path to the project directory (set to `projectDir.absolutePath`) |

## Compile-time errors and warnings

| Situation | Outcome |
|---|---|
| Feature file not found | **Error** with full resolved path |
| `@BehaveType` auto-detect on type with zero constructor params | **Error** |
| Two `@BehaveType` entries claim same placeholder or overlapping fields | **Error** |
| `@BehaveType` field auto-detect matches no step | **Warning**, fallback to `String` |
| Two steps normalise to the same step text with different param types | **Warning**, first wins |
| Two steps produce the same method name | Numeric suffix added, no error |
