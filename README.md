# kotlin-behave

BDD testing framework for Kotlin Multiplatform. Write Gherkin feature files, get
**type-safe step definitions generated at compile time** — the compiler tells you when a
step is missing, and the IDE navigates straight from Gherkin to code.

```gherkin
Feature: Todo list

  Scenario: Add a todo item
    When I add a todo "Buy groceries"
    Then the todo "Buy groceries" is displayed
```

```kotlin
@BehaveFeature("features/todo.feature")
class TodoSteps : TodoStepsSpec {                       // TodoStepsSpec is generated by KSP
    override suspend fun whenIAddATodo(string: String) { /* ... */ }
    override suspend fun thenTheTodoIsDisplayed(string: String) { /* ... */ }
}
```

That's the whole loop: **write the feature → annotate a class → implement the generated
interface**. KSP also generates the wiring and the Kotest test, so there's nothing else to
register.

## Typed step parameters

Mark a step parameter with `@Type` and it arrives as a domain type instead of a raw `String`.
A `@TypeConverter` function does the mapping — and it can absorb **any number** of step
tokens, not just one:

```gherkin
  Scenario: A custom board
    Given I have board size 8 10
    When Drop item type "square"
```

```kotlin
data class Size(val width: Int, val height: Int)

@TypeConverter
fun convertToSize(width: Int, height: Int) = Size(width, height)   // two {int} tokens → one Size

enum class Item { SQUARE, LINE, T, L /* ... */ }

@TypeConverter
fun toItem(name: String) = Item.valueOf(name.uppercase())          // one token → enum

@BehaveFeature("features/board.feature")
class BoardSteps : BoardStepsSpec {
    override suspend fun iHaveBoardSize(@Type(Size::class) size: Size) {   // 8 10 → Size(8, 10)
        board = Board(size.width, size.height)
    }
    override suspend fun dropItemType(@Type(Item::class) item: Item) {     // "square" → Item.SQUARE
        board.drop(item)
    }
}
```

The converter lives in the same compilation unit as the steps; plain and extension functions
both work. See [KSP Code Generation](docs/ksp.md#type--typeconverter) for `@Type`,
`@TypeConverter`, `@BehaveType` (multi-token / DataTable types) and `@BehaveCast`.

## Setup

Full setup — `:core` + `:kotest` + `:annotations` + `:ksp` for compile-time-safe steps:

```kotlin
// build.gradle.kts
plugins {
    id("com.google.devtools.ksp") version "2.1.20-1.0.32"
}

dependencies {
    testImplementation("io.mcol.kotlin-behave:core:0.1.0")
    testImplementation("io.mcol.kotlin-behave:kotest:0.1.0")
    testImplementation("io.kotest:kotest-runner-junit5:5.9.1")   // JVM
    compileOnly("io.mcol.kotlin-behave:annotations:0.1.0")
    kspTest("io.mcol.kotlin-behave:ksp:0.1.0")
}

ksp {
    arg("behave.featureDir", "src/test/resources")   // default: src/commonTest/resources
    arg("behave.projectDir", projectDir.absolutePath)
}

tasks.withType<Test> { useJUnitPlatform() }
```

**1 — Write the feature** (`src/test/resources/features/todo.feature`):

```gherkin
Feature: Todo list

  Background:
    Given the todo list is empty

  Scenario: Add a todo
    When I add a todo "Buy groceries"
    Then the todo "Buy groceries" is displayed
```

**2 — Annotate your class.** KSP generates the `TodoStepsSpec` interface, a
`val generatedTodoSteps` instance, and a `TodoGherkinTest` Kotest class:

```kotlin
@BehaveFeature("features/todo.feature")
class TodoSteps : TodoStepsSpec {
    private val todos = mutableListOf<String>()

    override suspend fun givenTheTodoListIsEmpty() { todos.clear() }
    override suspend fun whenIAddATodo(string: String) { todos.add(string) }
    override suspend fun thenTheTodoIsDisplayed(string: String) { check(string in todos) }
}
```

**3 — Run:**

```bash
./gradlew test
```

Add lifecycle hooks or a custom scenario runner by implementing extra interfaces on the
Steps class (`ScenarioHooks`, `ScenarioRunner`) — see [Kotest Integration](docs/kotest.md).
Prefer fewer dependencies? `:core` works standalone with a manual step DSL — see
[Core API](docs/core.md).

## Sharing steps across features

When the same step text appears in 2+ feature files, KSP **errors** unless you opt in —
silent duplication is never allowed. Choose one:

- **`@StepsMixin`** — write the step body once in an interface; matching specs inherit it.
- **`@DivergentStep`** — mark every override when the same step needs different bodies per
  feature (e.g. web seeds a cookie, mobile calls a debug login).

Full examples in [KSP Code Generation](docs/ksp.md#sharing-steps-across-features).

## Claude Code BDD skill

kotlin-behave ships a [Claude Code](https://claude.ai/code) plugin with a **BDD skill** that
guides AI agents through the correct workflow — feature file first, then steps, then
interfaces, then implementation. Install with `claude --add-dir /path/to/kotlin-behave`, then
invoke `/kotlin-behave:bdd`. Full content: [`skills/bdd/SKILL.md`](skills/bdd/SKILL.md).
